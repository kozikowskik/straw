# Support

## Questions

Open a GitHub issue for questions and usage help. Use the `question` label when it is available.

Include what you are trying to build, the Bubble Tea version you use, the `straw` version or commit, and a small code sample when possible.

## Bugs

Report reproducible bugs with a GitHub issue. Include your `straw` version, Bubble Tea version, Go version, operating system, and a small reproduction when possible.

## Feature Requests

Open a GitHub issue describing the use case, the current workaround, and why the feature belongs in `straw` rather than host application code.

## Security

Do not report security concerns in public issues. Use GitHub private vulnerability reporting:

https://github.com/kozikowskik/straw/security/advisories/new
